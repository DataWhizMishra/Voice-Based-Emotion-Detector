                      1. SYSTEM OVERVIEW
================================================================================

DeepEmotion AI is a real-time voice emotion detection system designed to identify
emotional distress in audio recordings. It combines a CNN-based emotion classifier
with a distress scoring engine, speech-to-text transcription, and sentiment analysis
to provide comprehensive emotional state assessment.

KEY FEATURES:
- Real-time voice recording and analysis
- File upload for batch processing (WAV, MP3, OGG, FLAC, M4A, WebM)
- 8-class emotion classification (neutral, calm, happy, sad, angry, fearful,
  disgust, surprised)
- Distress scoring (0-100) with 4 risk levels (low, medium, high, critical)
- Escalation detection (rising/decreasing emotional trends)
- Speech-to-text transcription via Whisper (OpenAI)
- Sentiment analysis via XLM-RoBERTa (cardiffnlp/twitter-xlm-roberta-base-sentiment)
- Hidden distress detection from speech content
- Browser notifications for critical/high risk alerts
- Session tracking and dashboard analytics

================================================================================
                 5. STEP-BY-STEP PROCESS WALKTHROUGH
================================================================================

SCENARIO: User uploads an audio file for full analysis
──────────────────────────────────────────────────────

STEP 1: User Interface
  • User navigates to "File Upload" tab
  • User drags and drops an audio file (or clicks to browse)
  • File validates: must be .wav, .mp3, .ogg, .flac, .m4a, or .webm
  • File size must be < 50MB (Flask MAX_CONTENT_LENGTH)
  • File Selected UI appears with filename and size

STEP 2: User clicks "Full Analysis"
  • Button enters loading state (spinner + disabled)
  • POST /api/analyze/full with FormData (audio file + language)

STEP 3: Backend receives file
  • request.files['audio'] retrieved
  • secure_filename applied to prevent path traversal
  • File saved to uploads/{timestamp}_{original_filename}

STEP 4: Format Conversion
  • Temporary .webm file created at temp path
  • convert_to_wav() called with ffmpeg path
  • ffmpeg -y -i {input} -ar 22050 -ac 1 {output}.wav
  • Validates: returncode==0 AND output exists AND size>0
  • If fails: error returned to user

STEP 5: Audio Properties
  • get_audio_properties(wav_path) using librosa
  • Returns: {duration, sample_rate, samples}

STEP 6: Emotion Prediction
  • extract_features(wav_path) → 40 MFCC features
  • If model_loaded:
    - scaler.transform(features.reshape(1,-1))
    - model.predict(features_scaled.reshape(1,40,1))
    - probabilities array → 8 emotion classes
  • If no model: random Dirichlet (demo mode)
  • Result: emotion name, confidence %, all probabilities, top3

STEP 7: Distress Scoring
  • DistressEngine.compute(probabilities_dict)
  • Each emotion weight × probability → sum × 100
  • Assigns risk_level: low/medium/high/critical
  • Sets alert_triggered if level in (high, critical)

STEP 8: Spectrogram Generation
  • SpectrogramGenerator.generate_spectrogram(wav_path):
    - librosa.load(wav_path, sr=22050)
    - librosa.display.waveshow → waveform subplot
    - librosa.feature.melspectrogram → mel spec subplot
    - plt.savefig(format='png', base64 encoded)
  • SpectrogramGenerator.generate_waveform(wav_path):
    - Same but single waveform subplot only

STEP 9: Speech-to-Text (if duration >= 1.5s and Whisper loaded)
  • stt_engine.transcribe(wav_path, language='en')
  • audio resampled to 16000Hz for Whisper
  • log_mel_spectrogram computed (n_mels from model.dims.n_mels)
  • Pad/truncate to model.dims.n_audio_ctx
  • model.decode(mel, language, task='transcribe')
  • _analyze_sentiment(transcribed_text):
    - _normalize_hinglish(text) — adds English equivalents for Hinglish
    - sentiment_analyzer(normalized_text[:512])
    - Returns label (positive/negative/neutral) + score
  • _detect_hidden_distress(text, sentiment):
    - Check keywords in: emergency, fear, danger, pain, panic categories
    - detected = len(indicators) > 0 OR sentiment['label']=='negative'

STEP 10: Session Update
  • session_tracker.add(result) — appends to history with timestamp
  • If alert_triggered: alert_manager.trigger(result)
  • Result JSON assembled and returned

STEP 11: Frontend Renders Result
  • showResult(data) called
  • --emotion-color CSS variable set on hero
  • Distress score and risk badge updated
  • All 8 probability bars animated (width: 0% → target%)
  • Audio info grid populated (duration, sample rate, source)
  • Top 3 predictions displayed
  • Spectrogram/waveform images shown (toggleable)
  • Transcript panel shown if transcript exists
  • Hidden distress alert shown if detected
  • Emergency banner shown if alert_triggered
  • Browser notification sent if permission granted
  • Result section scrolls into view

STEP 12: Cleanup
  • uploads/{timestamp}_{filename} deleted via os.remove
  • Temporary WebM and WAV files deleted in finally block

                      7. ML MODEL ARCHITECTURE
================================================================================

TRAINING (model_trainer.py):
────────────────────────────
Dataset: RAVDESS (Ryerson Audio-Visual Database of Emotional Speech and Song)
- 8 emotions: neutral, calm, happy, sad, angry, fearful, disgust, surprised
- File naming: {modality}-{channel}-{emotion}-{intensity}-{statement}-{repetition}-{actor}.wav

Training Process:
1. Load all .wav files from dataset path
2. For each file:
   - extract_features(file_path) → 40 MFCCs (kaiser_fast, offset=0.0)
   - augment_audio(file_path) → 4 augmented versions:
     * Time stretch: rate=0.9 and rate=1.1
     * Pitch shift: n_steps=-2 and n_steps=2
     * Add noise: noise_factor=0.005
   - So 1 file → 5 samples (1 original + 4 augmented)
3. Data split: 80% train, 20% test (stratified)
4. StandardScaler fit on X_train, transform X_test
5. Class weights computed for imbalanced data
6. One-hot encode labels (8 classes)
7. Reshape to (samples, 40, 1) for CNN

Model Architecture (build_model):
┌─────────────────────────────────────────────────────────────────┐
│  Input: (40, 1)  — 40 MFCC features, 1 channel                   │
│                                                                 │
│  Block 1:                                                       │
│    Conv1D(256, kernel=3, padding='same') + L2(1e-4)            │
│    BatchNormalization                                          │
│    Activation('relu')                                          │
│    Dropout(0.3)                                                │
│    MaxPooling1D(pool_size=2)                                   │
│                                                                 │
│  Block 2:                                                       │
│    Conv1D(128, kernel=3, padding='same') + L2(1e-4)            │
│    BatchNormalization                                          │
│    Activation('relu')                                          │
│    Dropout(0.3)                                                │
│    MaxPooling1D(pool_size=2)                                   │
│                                                                 │
│  Block 3:                                                       │
│    Conv1D(64, kernel=3, padding='same') + L2(1e-4)             │
│    BatchNormalization                                          │
│    Activation('relu')                                          │
│    Dropout(0.3)                                                │
│                                                                 │
│  Flatten                                                        │
│  Dense(128) + L2(1e-4) + BatchNorm + Activation('relu') + Dropout(0.4)
│  Dense(8, activation='softmax')                                │
└─────────────────────────────────────────────────────────────────┘

Training Config:
  • Optimizer: Adam (lr=0.001)
  • Loss: categorical_crossentropy
  • Batch size: 64
  • Max epochs: 100
  • EarlyStopping: patience=15, restore_best_weights
  • ReduceLROnPlateau: factor=0.5, patience=5, min_lr=1e-6
  • Validation split: 10%

Output: Emotion_Voice_Detection_Model.h5 + scaler.pkl

INFERENCE (app.py — simplified model):
┌─────────────────────────────────────────────────────────────────┐
│  Input: (40, 1)  — MFCC features from librosa                   │
│                                                                 │
│  Conv1D(128, kernel=5, padding='same')                         │
│  Activation('relu')                                            │
│  Dropout(0.1)                                                  │
│  MaxPooling1D(pool_size=8, strides=8)                         │
│                                                                 │
│  Conv1D(128, kernel=5, padding='same')                         │
│  Activation('relu')                                            │
│  Dropout(0.1)                                                  │
│                                                                 │
│  Flatten                                                        │
│  Dense(8, activation='softmax')                                │
└─────────────────────────────────────────────────────────────────┘

NOTE: The inference model in app.py is a SIMPLIFIED version of the training
model. The full training model has 3 conv blocks + batch norm + dense layers.
The inference model in app.py only has 2 conv blocks without batch norm.
This is the deployed inference architecture.

EMOTION CLASSES (index → label):
  0: neutral   4: angry
  1: calm       5: fearful
  2: happy      6: disgust
  3: sad        7: surprised

DISTRESS WEIGHTS (emotion → weight):
  fearful:  1.0  (highest — primary distress indicator)
  angry:    0.7
  sad:      0.6
  disgust:  0.4
  surprised:0.2
  neutral:  0.0
  calm:     0.0
  happy:    0.0

DISTRESS THRESHOLDS:
  low:      < 40
  medium:   40–59
  high:     60–79
  critical: >= 80
